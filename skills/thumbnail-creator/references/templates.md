# Thumbnail Prompt Templates

These are the exact prompt templates. Fill every placeholder before sending to nano banana.

---

## Base Prompt (layer on top of any template for consistency)

```
Create a 16:9 blog thumbnail in [BRAND STYLE] with [PRIMARY COLOR] + [ACCENT COLOR], high contrast, minimalism, slight grain texture, cinematic lighting, no watermark, no text, clean composition with one focal subject. Theme: [TOPIC] using [TOOL/TECH]. Emphasize: [KEY SYMBOL] to represent the idea.
```

**Default values if not specified by user:**
- [BRAND STYLE] = "retro collage meets futuristic blueprint"
- [PRIMARY COLOR] = deep navy (#0A1628)
- [ACCENT COLOR] = electric amber (#F59E0B)

## Template 1: Object Metaphor

**Use for:** How-to tools, tutorials, frameworks, rebuild stories, concept explanations.

```
16:9 thumbnail. Central object: [KEY SYMBOL] representing [TOPIC]. Style: [BRAND STYLE], smooth gradients with grain, bold shadows, ultra-readable even at small size, minimal background elements. Atmosphere: [MOOD]. No text.
```

**Tips for filling:**
- [KEY SYMBOL] should be one specific, tangible object — not an abstract concept. "A glowing hourglass with gears inside" not "the concept of time and engineering"
- Keep background minimal: solid color, subtle gradient, or faint grid lines
- The object should be centered and occupy 40-60% of the frame

## Template 2: Neoclassical Hero

**Use for:** AI + human collaboration, writing, decision frameworks, leadership topics.

```
16:9 thumbnail. Classical statue/engraving of [ROLE], recolored in [PRIMARY COLOR]+[ACCENT COLOR], wearing [MODERN PROP], holding/looking at [KEY SYMBOL] tied to [TOOL/TECH]. Style: retro-print halftone + modern color blocking, clean, editorial. No text.
```

**Tips for filling:**
- [ROLE] choices: writer, engineer, manager, architect, analyst, philosopher, strategist
- [MODERN PROP] choices: headphones, VR goggles, smartwatch, holographic display, wireless earbuds, sunglasses
- The statue should feel dignified but not stiff — a slight lean or gesture adds life
- The color blocking should be bold: statue in one tone, background in another, key symbol in accent color

## Template 3: Duotone Portrait + Pattern

**Use for:** Interviews, opinion pieces, personal experience posts, "from the trenches" content.

```
16:9 thumbnail. Duotone portrait of [PERSON/ARCHETYPE], background pattern built from [KEY SYMBOL] repeated, brand color palette [PRIMARY COLOR]+[ACCENT COLOR]+[NEUTRAL], sharp lighting, magazine cover feel. Optional tiny logo placeholder in corner. No text.
```

**Tips for filling:**
- [PERSON/ARCHETYPE]: "a focused engineer in profile", "a contemplative builder looking up", "a determined analyst with crossed arms"
- The portrait should be stylized/duotone, NOT photorealistic
- Background pattern: the key symbol repeated at 20-30% opacity creates texture without competing with the portrait
- Neutral color: warm gray (#D4D0C8) for backgrounds or tertiary elements

## Template 4: Blueprint/Engineering

**Use for:** Architecture, systems design, workflows, reverse engineering, technical teardowns.

```
16:9 thumbnail. Style: blueprint/technical diagram, monochrome base with [ACCENT COLOR] highlights. Elements: wireframe of [TOOL/TECH], oversized schematic of [KEY SYMBOL], annotation-style arrows, clean, precise. No clutter. No text.
```

**Tips for filling:**
- Base should be deep navy or dark blue-black, mimicking a real blueprint
- [ACCENT COLOR] highlights on the KEY elements only — not everything
- Annotation arrows should be minimal (2-3 max) pointing to the interesting parts
- The wireframe should be simplified — suggest the tool, don't recreate its exact UI
- Works especially well when the KEY SYMBOL is shown in "exploded view" or "x-ray" style

---

## Template 5: App Grid Series

**Use for:** Tool stacks, multi-tool roundups, productivity setups, "my setup" posts.

```
16:9 thumbnail. Row of rounded-square icons (3-6) showing: [TOOL/TECH ICONS]. One icon enlarged for focus. Background: [PRIMARY COLOR] block with subtle texture. Style: modern UI, minimal, bold edges. No text.
```

**Tips for filling:**
- Icons should be ABSTRACT representations of the tools — not actual logos (to avoid trademark issues)
- Abstract representations: terminal icon for CLI tools, brain icon for AI, gear for automation, database cylinder for data tools, rocket for deployment
- One icon should be 1.5-2x larger than the others to create hierarchy
- Background texture: subtle noise, faint dot grid, or very soft gradient
- 3-4 icons is ideal for readability at small sizes; 6 is the max
